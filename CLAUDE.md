# Project preferences

- Animation pacing: after each scene's closing beat (KPI flip, payoff line, final state), always leave ~1.5–2s of settled hold time before cutting to the next scene. The user reads every frame — never end a scene right as its last element finishes animating. When adding content to a scene, extend the scene duration rather than compressing holds.
- Pasted SVG mockups from the user often arrive truncated — rebuild the UI natively in the animation's style instead of embedding, and say so.
- Product-UI scenes should reuse the established card style: white card, Perry logo + mono section label header, serif titles (Source Serif 4 semibold), IBM Plex Mono metadata, accent #009C7F.
